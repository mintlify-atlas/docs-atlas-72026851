This guide walks you through getting Movie Circle running on your machine. You'll clone the repository, configure environment variables, apply database migrations, and start the development server. If you prefer containers, there's also a Docker Compose path that handles the database for you.

## Prerequisites

Before you start, make sure you have the following installed:

- **Node.js** v18 or later
- **pnpm** — install with `npm install -g pnpm`
- **PostgreSQL** 14 or later (if running manually), or **Docker** and **Docker Compose** (if using the container path)
- A **TMDB API key** — register at [themoviedb.org](https://www.themoviedb.org/settings/api) and create a free API key

<Steps>
  <Step title="Clone the repository">
    Clone the Movie Circle repository and move into the project directory:

    ```bash
    git clone https://github.com/adrian-237/movie-circle.git
    cd movie-circle
    ```
  </Step>

  <Step title="Install dependencies">
    Install all Node.js dependencies using pnpm:

    ```bash
    pnpm install
    ```
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root and populate it with your configuration. See the [environment variables reference](/environment-variables) for a full description of each variable and where to get the required credentials.

    At minimum, you need `DATABASE_URL`, `NEXT_PUBLIC_TMDB_API_KEY`, `TMDB_API_KEY`, `NEXTAUTH_URL`, and `NEXTAUTH_SECRET` before the app will start.

    <Tip>
      Generate a strong `NEXTAUTH_SECRET` by running `openssl rand -base64 32` in your terminal.
    </Tip>
  </Step>

  <Step title="Set up the database">
    Choose the path that matches your setup: run PostgreSQL locally, or spin it up with Docker Compose.

    <CodeGroup>

    ```bash Manual (local PostgreSQL)
    # Apply migrations and create the schema
    pnpm migrate:dev

    # Seed the database with sample data
    pnpm db:seed
    ```

    ```bash Docker Compose
    # Start a PostgreSQL container in the background
    docker-compose up -d

    # Apply migrations once the container is healthy
    pnpm migrate:dev

    # Seed the database
    pnpm db:seed
    ```

    </CodeGroup>

    <Note>
      The `docker-compose.yml` starts only the PostgreSQL service — it does not run the Next.js app. The app runs on your host via `pnpm dev`. Set `DATABASE_URL` to `postgresql://postgres:postgres@localhost:5432/movie_circle` to match the container's default credentials.
    </Note>
  </Step>

  <Step title="Start the development server">
    Start Movie Circle in development mode:

    ```bash
    pnpm dev
    ```

    Open [http://localhost:3000](http://localhost:3000) in your browser. You should see the Movie Circle landing page.
  </Step>
</Steps>

## Create your first collection

Once the app is running and you've signed up:

1. Click **Sign up** and create an account with an email and password (or use GitHub/Google OAuth if you've configured those providers).
2. After sign-in, three default collections — **To Watch**, **Watching**, and **Watched** — are created automatically on your account.
3. Navigate to **Discover** to browse trending movies and TV shows.
4. Open any title, click **Add to collection**, and choose or create a collection. You can optionally add a personal rating and notes.
5. Go to **Collections** to view your lists and share them via a public link.

<Tip>
  You can create additional collections from the Collections page — for example, a "Favorites" list or a themed watch party collection.
</Tip>
